## 1. CRISPE 产品需求

### 1.1 Context：产品背景

《日报阅读器》v1.0 是一个面向 iOS 学习作品集的中文日报阅读 App。产品以 SwiftUI 为主要技术方向，但本文档只描述产品需求，不包含代码实施方案。

产品目标是提供一个轻量、清晰、可失败恢复的日报阅读体验。用户可以打开 App 浏览今日内容，进入文章详情阅读，在网络异常时查看可用缓存，并通过系统分享面板分享文章链接。

本产品直连候选知乎日报接口作为内容来源。该接口不是已确认的官方开放契约，可能存在变更、限流、不可用、内容授权不明确等风险。因此 v1.0 只定位为学习作品集，不作为商业产品或正式发布产品。

### 1.2 Request：产品请求

v1.0 要完成一个“阅读闭环”，让用户可以完成以下路径：

1. 打开 App；
2. 查看今日日报首页；
3. 浏览顶部故事横向卡片；
4. 浏览普通日报列表；
5. 加载历史日报；
6. 点击文章进入详情页；
7. 阅读文章内容；
8. 使用系统分享功能分享文章链接；
9. 在无网络或接口失败时查看可用缓存；
10. 在无缓存时看到明确中文错误提示和重试入口。

v1.0 不追求完整复刻知乎日报官方 App，而是实现一个现代 iOS 原生风格的日报阅读器。

### 1.3 Input：输入与数据来源

#### 业务输入

- 产品名称：日报阅读器
- 产品版本：v1.0
- 产品定位：学习作品集
- 目标用户：希望体验简洁日报阅读流程的普通用户，以及用于展示 SwiftUI 产品实现能力的评审者
- 主要语言：中文

#### 内容输入

候选接口如下：

| 场景 | 接口 |
| --- | --- |
| 获取今日日报 | `GET https://news-at.zhihu.com/api/4/news/latest` |
| 获取历史日报 | `GET https://news-at.zhihu.com/api/4/news/before/{yyyymmdd}` |
| 获取文章详情 | `GET https://news-at.zhihu.com/api/4/news/{id}` |

接口字段需要按不稳定外部数据处理。任何字段都不能默认一定存在，尤其是图片、正文、分享链接、顶部故事等内容。

#### 体验输入

- 首页顶部故事使用横向卡片，不做自动轮播。
- 文章详情允许使用 WebView 类阅读容器渲染 HTML。
- 深色模式跟随系统。
- 缓存范围为最近 30 天列表与已打开文章详情。
- 图片可缓存，但不作为完整离线阅读承诺。

### 1.4 Step：用户路径

#### 主路径：正常阅读

1. 用户打开 App。
2. App 展示首页加载态。
3. 加载成功后，首页展示今日日期、顶部故事横向卡片、日报列表。
4. 用户点击顶部故事或普通列表文章。
5. App 进入文章详情页。
6. 详情页展示文章标题、头图（如有）、正文内容。
7. 用户点击分享按钮。
8. App 调起系统分享面板，分享文章标题与链接。
9. 用户返回首页。
10. 用户继续向下加载历史日报。

#### 异常路径：无网络但有缓存

1. 用户打开 App 时网络不可用。
2. App 检查本地缓存。
3. 如果存在最近 30 天内可用列表缓存，则展示缓存列表。
4. 首页顶部提示“当前离线，正在显示缓存内容”。
5. 用户点击已缓存文章。
6. 如果文章详情已缓存，则展示缓存详情。
7. 如果文章详情未缓存，则展示详情错误态。

#### 异常路径：无网络且无缓存

1. 用户打开 App 时网络不可用。
2. App 未找到可用首页缓存。
3. 首页展示错误态。
4. 错误态文案提示“网络不可用，请检查连接后重试”。
5. 用户可以点击“重试”。

#### 异常路径：接口返回异常

1. App 请求接口失败、超时、返回非预期数据或解析失败。
2. 如果已有旧内容，保留旧内容，不清空页面。
3. 页面展示轻提示：“刷新失败，已保留上次内容”。
4. 如果没有可用旧内容，则展示完整错误态。

### 1.5 Constraint：约束

#### 产品约束

- v1.0 不做官方视觉 1:1 复刻。

#### 功能约束

v1.0 只做阅读闭环，不做复杂社区或账号能力。

明确不做：

- 登录
- 注册
- 评论
- 点赞
- 收藏同步
- 本地收藏夹
- 搜索
- 主题日报
- 栏目页
- 推送通知
- 小组件
- iPad 适配
- 横屏适配
- 多窗口
- App Store 上架流程
- 官方知乎日报视觉复刻
- 完整离线图片阅读
- 批量抓取历史文章

#### 数据约束

- 数据源为候选非官方接口。
- 接口可能变更、限流或失效。
- 接口返回字段可能缺失、为空或类型变化。
- 产品必须允许接口失败，并提供可恢复体验。

- 图片缓存不纳入完整离线承诺。
- 缓存损坏不能导致 App 白屏或崩溃。
- 网络失败不能覆盖可用缓存。

### 1.6 Example：使用示例

#### 示例 1：日常阅读

用户早上打开《日报阅读器》，首页展示今日内容。用户横向浏览顶部故事，点击感兴趣的文章进入详情页阅读。读完后，用户点击分享按钮，将文章链接通过系统分享面板发送给朋友。

#### 示例 2：地铁离线阅读

用户晚上在地铁中打开 App，此时网络不可用。App 展示上次成功加载的首页缓存，并提示“当前离线，正在显示缓存内容”。用户点击之前打开过的文章，可以继续查看缓存详情。

#### 示例 3：接口失败但保留旧内容

用户下拉刷新首页时接口请求失败。App 不清空当前列表，而是在保留旧内容的基础上提示“刷新失败，已保留上次内容”。用户仍可继续阅读已有内容。

## 2. v1.0 功能范围

### 2.1 首页日报

首页是 v1.0 的主入口。

首页需要展示：

- 今日日期；
- 顶部故事横向卡片；
- 普通日报列表；
- 日期分组；
- 下拉刷新；
- 加载历史日报；
- 加载态；
- 空态；
- 错误态；
- 离线缓存态。

首页不做：

- 自动轮播；
- 搜索；
- 主题频道；
- 个性化推荐；
- 登录入口。

#### 首页验收标准

- 有网络时首次打开可展示今日内容。
- 无网络且有缓存时展示缓存列表。
- 无网络且无缓存时展示错误态和重试入口。
- 下拉刷新失败不清空旧内容。
- 历史加载失败不影响已有列表。
- 缺图文章仍能正常展示。
- 深色模式下文字和控件可读。

### 2.2 顶部故事

顶部故事采用横向卡片，不采用自动轮播。

卡片内容：

- 封面图；
- 标题；
- 点击进入详情。

异常处理：

- 图片缺失时展示默认占位；
- 标题过长时最多显示 2 行；
- 接口没有顶部故事时隐藏顶部故事区；
- 隐藏顶部故事区不能影响普通列表展示。

### 2.3 普通日报列表

普通日报列表展示当天或历史日期下的文章。

列表项内容：

- 标题；
- 缩略图（如有）；
- 来源/提示文案（如接口提供）；
- 点击进入详情。

异常处理：

- 无图文章使用纯文本卡片或占位图；
- 图片加载失败不能影响标题阅读；
- 缺失标题的文章不应展示为可点击正常文章；
- 重复文章需要去重，避免历史加载时重复出现。

### 2.4 历史日报

用户可以加载更早日期的日报。

要求：

- 按日期分组展示；
- 支持向前加载历史；
- 加载中显示底部 loading；
- 加载失败时保留已有列表；
- 加载失败时提供重试入口；
- 不承诺无限历史完整性。

历史日报不做：

- 日历选择器；
- 指定日期跳转；
- 历史搜索；
- 批量预加载历史文章。

### 2.5 文章详情

文章详情用于阅读单篇文章。

详情页包含：

- 返回按钮；
- 文章标题；
- 头图（如接口提供）；
- HTML 正文；
- 分享按钮；
- 加载态；
- 错误态；
- 缓存态。

异常处理：

- `body` 为空：展示“文章内容暂不可用”；
- 详情请求失败且有缓存：展示缓存详情；
- 详情请求失败且无缓存：展示错误页和重试按钮；
- 缺少头图：隐藏头图区域或展示轻量占位；
- 分享链接缺失：禁用分享按钮或提示“当前文章暂不可分享”；
- Web 内容加载失败：展示可恢复错误态；
- 外链打开不能破坏返回路径。

#### 详情验收标准

- 点击文章可进入详情页。
- HTML 正文可阅读。
- 详情失败有明确中文错误提示。
- 已缓存文章断网可回看。
- 缺正文不白屏。
- 缺分享链接不产生错误分享。
- 深色模式下详情页基本可读。

### 2.6 分享

分享使用系统分享面板。

分享内容优先级：

1. 详情接口 `share_url`；
2. 列表接口中的文章 URL；
3. 若都不存在，则不可分享。

分享内容：

- 文章标题；
- 文章链接。

v1.0 不做：

- 分享海报；
- 图片合成；
- 自定义分享渠道；
- 微信 SDK；
- 微博 SDK。

#### 分享验收标准

- 正常文章可以调起系统分享面板。
- 分享内容不能为空。
- 缺失分享链接时不能分享错误内容。
- 分享失败不能影响当前阅读。

### 2.7 缓存

缓存用于提升弱网与离线场景体验。

缓存范围：

- 最近 30 天日报列表；
- 用户打开过的文章详情；
- 图片由图片加载能力处理，不纳入完整离线阅读承诺。

缓存策略：

- 有网络时优先请求网络；
- 请求成功后写入缓存；
- 请求失败时读取缓存；
- 无缓存时展示错误态；
- 刷新失败不能清空已有内容；
- 缓存数据需要在 UI 或状态中与实时网络数据区分。

缓存提示：

- 离线且有缓存：首页提示“当前离线，正在显示缓存内容”；
- 刷新失败但有旧内容：提示“刷新失败，已保留上次内容”；
- 离线且详情有缓存：详情页提示“正在显示缓存内容”；
- 离线且详情无缓存：展示详情错误态。

#### 缓存验收标准

- 最近 30 天列表可缓存。
- 已打开文章详情可缓存。
- 杀进程重启后仍能读取缓存。
- 缓存损坏不导致 App 崩溃或白屏。
- 网络失败不能覆盖可用缓存。

### 2.8 深色模式

v1.0 深色模式跟随系统。

要求：

- 首页在深色模式下可读；
- 列表卡片在深色模式下可读；
- 错误态和空态在深色模式下可读；
- 详情页外壳在深色模式下可读；
- Web 内容至少不得出现明显不可读问题。

v1.0 不做：

- App 内手动切换浅色/深色；
- 自定义主题色切换；
- 多套主题。

## 7. 产品风险

### 7.1 高风险

#### 候选接口非官方契约

`news-at.zhihu.com` 候选接口未被确认是稳定官方开放接口，可能出现字段变化、限流、403、接口下线等问题。

影响：

- 首页可能无法加载；
- 历史日报可能无法加载；
- 文章详情可能无法展示；
- 分享链接可能缺失。

应对：

- 文档明确学习作品集定位；
- UI 必须覆盖错误态；
- 缓存必须支持失败兜底；
- 后续若要正式发布，应考虑自建代理或合法授权数据源。

#### 内容授权不明确

文章标题、正文、图片来自第三方内容源。学习用途不等于商业授权。

影响：

- 不适合商用；
- 不适合直接上架 App Store；
- 不适合使用知乎官方品牌资产。

应对：

- 产品名使用“日报阅读器”；
- 不使用知乎官方 Logo；
- 文档中明确非官方学习项目；
- README 中应保留风险声明。

#### Web 内容不可控

文章详情为 HTML 内容，可能包含远端图片、CSS、外链或不稳定结构。

影响：

- 深色模式可能不完美；
- 字体与排版可能不可控；
- 外链行为需要兜底；
- 正文为空时可能白屏。

应对：

- 详情页必须有 Web 内容失败态；
- `body` 为空必须有中文提示；
- 外链不能破坏返回路径。

### 7.2 中风险

#### 缓存掩盖接口失败

如果只展示缓存，不提示数据来源，用户和测试者可能误以为接口成功。

应对：

- 离线缓存必须提示；
- 刷新失败必须提示；
- 产品状态需要区分网络数据与缓存数据。

#### 历史日报日期语义误解

历史接口通常表示“获取指定日期之前的日报”，而不是“获取指定日期当天日报”。

应对：

- 产品文档中避免承诺指定日期跳转；
- v1.0 只做向前加载历史。

#### 图片加载失败

图片可能为空、失效或加载很慢。

应对：

- 图片失败不能影响文字阅读；
- 缺图文章仍可点击进入详情。

## 8. 验收标准

### 8.1 主路径验收

v1.0 必须完成：

- 用户打开 App 后可以看到首页；
- 首页可以展示今日内容；
- 首页可以展示顶部故事横向卡片；
- 首页可以展示普通日报列表；
- 用户可以进入文章详情；
- 用户可以阅读文章正文；
- 用户可以分享文章链接；
- 用户可以返回首页；
- 用户可以继续加载历史日报。

### 8.2 异常路径验收

v1.0 必须完成：

- 无网络且有缓存时展示缓存内容；
- 无网络且无缓存时展示错误态；
- 刷新失败时不清空已有内容；
- 历史加载失败时不影响已有列表；
- 详情失败时有明确中文错误提示；
- 缺正文时不白屏；
- 缺分享链接时不分享错误内容；
- 缓存损坏时不导致 App 崩溃或白屏。

### 8.3 范围验收

v1.0 不应出现：

- 登录入口；
- 评论入口；
- 点赞入口；
- 搜索入口；
- 主题日报入口；
- 收藏同步能力；
- iPad 专属布局；
- 官方知乎日报 Logo；
- App Store 上架承诺。

## 9. 完成定义

产品需求 v1.0 视为完成，当且仅当：

- CRISPE 六项均已覆盖；
- v1.0 功能范围清晰；
- v1.0 不做项清晰；
- 后续版本规划清晰；
- 首页、详情、分享、缓存、深色模式均有验收标准；
- 接口风险、学习作品集定位、非官方数据源说明明确；
- 文档没有混入代码实施细节。
