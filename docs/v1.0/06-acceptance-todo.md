# 《日报阅读器》v1.0 分角色验收 TODO 清单

本文档由 `01-product-requirements.md` 的验收标准与 `05-acceptance-plan-D.md` 的验收角色整理而来，用于开发完成后的分工验收。

角色边界：

- 产品经理 agent：验收需求范围、主路径体验、中文文案与“不做项”。
- iOS 开发 agent：提供可验收构建、解释实现、修复缺陷，不做最终裁判。
- 测试 agent：执行 XCTest / XCUITest / 手工测试，提交测试证据。
- 监控/集成 agent：汇总构建、测试、改动范围、风险和验收材料。
- 监工验收 agent：独立复核最终结果，输出通过 / 有条件通过 / 不通过。

## 1. 产品经理 agent 验收 TODO

### 1.1 主路径与产品体验

- [ ] App 启动后可以进入首页。
- [ ] 首页在有网络时可以展示今日内容。
- [ ] 首页可以展示顶部故事横向卡片。
- [ ] 首页可以展示普通日报列表。
- [ ] 用户点击顶部故事可以进入文章详情页。
- [ ] 用户点击普通日报文章可以进入文章详情页。
- [ ] 详情页可以展示并阅读 HTML 正文。
- [ ] 用户可以从详情页返回首页。
- [ ] 用户可以继续加载历史日报。
- [ ] 历史日报按日期分组展示。
- [ ] 正常文章可以调起系统分享面板。
- [ ] 分享内容包含文章标题。
- [ ] 分享内容包含有效文章链接。

### 1.2 异常体验与中文文案

- [ ] 无网络且有缓存时，首页提示“当前离线，正在显示缓存内容”。
- [ ] 无网络且无缓存时，首页展示错误态和重试入口。
- [ ] 下拉刷新失败但有旧内容时，提示“刷新失败，已保留上次内容”。
- [ ] 详情失败时有明确中文错误提示。
- [ ] 详情请求失败且有缓存时，提示“正在显示缓存内容”。
- [ ] `body` 为空时展示“文章内容暂不可用”。
- [ ] 分享链接缺失时禁用分享按钮或提示“当前文章暂不可分享”。
- [ ] 缺正文、缺图、缺分享链接时均有合理降级体验。

### 1.3 范围边界

- [ ] v1.0 不出现登录入口。
- [ ] v1.0 不出现评论入口。
- [ ] v1.0 不出现点赞入口。
- [ ] v1.0 不出现搜索入口。
- [ ] v1.0 不出现主题日报入口。
- [ ] v1.0 不出现收藏同步能力。
- [ ] v1.0 不实现 iPad 专属布局。
- [ ] v1.0 不使用官方知乎日报 Logo。
- [ ] v1.0 不承诺 App Store 上架。
- [ ] v1.0 不实现自动轮播。
- [ ] v1.0 不实现日历选择器。
- [ ] v1.0 不实现指定日期跳转。
- [ ] v1.0 不实现历史搜索。
- [ ] v1.0 不批量预加载历史文章。
- [ ] v1.0 不实现分享海报。
- [ ] v1.0 不实现图片合成。
- [ ] v1.0 不接入微信 SDK。
- [ ] v1.0 不接入微博 SDK。
- [ ] v1.0 不实现自定义分享渠道。

## 2. iOS 开发 agent 准入自检 TODO

### 2.1 构建与运行准入

- [ ] 有可运行的 SwiftUI App。
- [ ] 提供可复现构建命令。
- [ ] 提供指定验收 commit。
- [ ] 说明 Xcode 版本、iOS 版本、设备或模拟器信息。
- [ ] 验收前无已知 P0 缺陷。
- [ ] Mock 或真实接口配置明确。

### 2.2 功能实现自检

- [ ] 首页支持加载今日内容。
- [ ] 首页支持顶部故事横向卡片。
- [ ] 首页支持普通日报列表。
- [ ] 首页支持下拉刷新。
- [ ] 首页支持加载历史日报。
- [ ] 历史加载中显示底部 loading。
- [ ] 历史加载失败时保留已有列表。
- [ ] 历史加载失败时提供重试入口。
- [ ] 接口没有顶部故事时隐藏顶部故事区。
- [ ] 隐藏顶部故事区不能影响普通列表展示。
- [ ] 顶部故事标题过长时最多显示 2 行。
- [ ] 缺失标题的文章不展示为可点击正常文章。
- [ ] 历史加载时重复文章需要去重。
- [ ] 详情页支持返回、标题、头图、HTML 正文、分享按钮、加载态、错误态、缓存态。
- [ ] Web 内容加载失败时展示可恢复错误态。
- [ ] 外链打开不能破坏详情页返回路径。

### 2.3 缓存与降级实现自检

- [ ] 最近 30 天日报列表可以缓存。
- [ ] 用户打开过的文章详情可以缓存。
- [ ] 有网络时优先请求网络。
- [ ] 请求成功后写入缓存。
- [ ] 请求失败时读取缓存。
- [ ] 请求失败且无缓存时展示错误态。
- [ ] 刷新失败不能清空已有内容。
- [ ] 缓存数据需要在 UI 或状态中与实时网络数据区分。
- [ ] 杀进程重启后仍能读取首页缓存。
- [ ] 杀进程重启后仍能读取已打开文章详情缓存。
- [ ] 缓存损坏不导致 App 崩溃。
- [ ] 缓存损坏不导致白屏。
- [ ] 网络失败不能覆盖可用缓存。
- [ ] 图片缓存不作为完整离线阅读承诺。
