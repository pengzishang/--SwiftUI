# 验收方案 D：《日报阅读器》v1.0

## 0. 说明

本文档用于指导《日报阅读器》v1.0 工程实施完成后的验收工作，重点解决三件事：

1. 明确谁来验收；
2. 明确按什么标准验收；
3. 明确如何留下可复查的验收证据。

本方案与以下文档配套使用：

- `01-product-requirements.md`：产品需求来源；
- `02-implementation-plan-A.md`：实施与多 agent 协作方案；
- `03-api-document-B.md`：接口契约与容错规则；
- `04-test-cases-C.md`：测试用例全集。

当前仓库尚未包含 SwiftUI 工程，因此本验收方案面向后续工程落地阶段。

## 1. 验收目标

v1.0 验收目标不是复刻官方知乎日报，而是确认 App 完成 PRD 定义的“阅读闭环”：

```text
启动 App
→ 查看今日日报首页
→ 浏览顶部故事和普通列表
→ 加载历史日报
→ 进入文章详情
→ 阅读 HTML 正文
→ 调起系统分享
→ 网络异常时可恢复或清晰失败
```

验收结论分为三类：

| 结论 | 含义 |
| --- | --- |
| 通过 | P0 全部通过，P1 无阻塞问题，允许进入下一阶段 |
| 有条件通过 | P0 全部通过，存在少量 P1/P2 问题，但不影响主链路和异常兜底 |
| 不通过 | 任一 P0 失败，或存在崩溃、白屏、数据误导、范围失控等严重问题 |

## 2. 验收角色与职责

| 角色 | 职责 | 是否允许修代码 |
| --- | --- | --- |
| 产品经理 agent | 判断功能是否符合 PRD、文案是否符合中文产品体验 | 否 |
| iOS 开发 agent | 提供构建包、修复验收缺陷、解释实现细节 | 是 |
| 测试 agent | 基于测试用例 C 编写并执行 XCTest 白盒单元测试、XCUITest 黑盒 UI 测试，提交测试报告 | 可写测试代码，不写业务代码 |
| 监控/集成 agent | 汇总构建、测试、改动范围和风险 | 可修集成问题 |
| 监工验收 agent | 独立做最终验收，输出通过/不通过结论 | 否 |

监工验收 agent 必须保持独立：

- 不参与业务代码开发；
- 不替开发 agent 修 bug；
- 不以“开发自测通过”作为最终依据；
- 必须基于可运行 App、测试结果和文档逐项验收。

## 4. 验收准入标准

进入正式验收前，必须满足：

- 工程可正常 build；
- App 可在 iOS 17+ 模拟器或真机启动；
- 首页、详情、分享主链路已实现；
- 网络 mock 或可控测试环境已准备；
- XCTest 单元测试能运行；
- XCUITest UI 测试能运行；
- P0 用例已有自动化覆盖；无法自动化的系统能力必须有手工替代说明；
- TDD 记录可追溯：关键 P0 功能能对应到先写的失败测试和最终通过结果；
- 无未说明的大范围未提交代码；
- 明确不做项没有作为功能入口出现。

若任一准入项不满足，不进入正式验收，先退回开发。

## 5. 验收范围

### 5.1 必验范围

| 模块 | 必验内容 |
| --- | --- |
| 首页 | 加载态、今日内容、顶部故事、普通列表、下拉刷新、错误态、空态、离线态 |
| 历史日报 | 向前加载、日期分组、失败保留已有内容、重复去重 |
| 文章详情 | 标题、头图、HTML 正文、加载态、错误态、缓存态 |
| 分享 | 系统分享面板、分享链接优先级、缺链接降级 |
| 缓存 | 最近 30 天列表、已打开详情、缓存损坏兜底、网络失败不覆盖缓存 |
| 接口容错 | 非 2xx、超时、破损 JSON、缺字段、空数组 |
| 图片 | 缺图、图片 404、慢加载不影响文字阅读 |
| 深色模式 | 首页、列表、错误态、详情外壳可读 |
| 范围控制 | 不出现登录、评论、点赞、收藏同步、搜索、主题日报、官方 Logo |

### 5.2 不作为 v1.0 阻塞的范围

以下内容不进入 v1.0 阻塞验收：

- iPad 专属布局；
- 横屏专属布局；
- App Store 上架流程；
- 微信/微博 SDK 分享；
- 分享海报；
- 登录、评论、点赞、收藏同步；
- 完整离线图片阅读；
- 批量抓取历史文章；
- 官方知乎日报视觉 1:1 复刻。

## 6. 验收流程

### Step 1：静态验收

检查文档和代码范围：

- PRD 中明确不做项没有进入 UI；
- 接口域名和路径符合接口文档 B；
- 模型解析允许字段缺失；
- 缓存状态可区分网络数据和缓存数据；
- 不使用知乎官方 Logo 或强品牌化视觉；
- 没有硬编码测试数据冒充真实接口结果。

输出：

```text
静态验收结论：通过 / 不通过
问题清单：
```

### Step 2：构建验收

推荐命令：

```text
xcodebuild -list
xcodebuild -project 知乎日报-SwiftUI.xcodeproj -scheme DailyReader -destination 'platform=iOS Simulator,name=iPhone 15' build
```

如工程采用 workspace 或不同 scheme，以实际工程为准，但必须记录命令。

通过标准：

- Debug 构建通过；
- 无阻塞级编译警告；
- App 可启动；
- 无启动即崩溃。

### Step 3：自动化测试验收

推荐命令：

```text
xcodebuild -project 知乎日报-SwiftUI.xcodeproj -scheme DailyReader -destination 'platform=iOS Simulator,name=iPhone 15' test
```

通过标准：

- P0 XCTest 白盒单元测试全部通过；
- P0 XCUITest 黑盒 UI 测试全部通过，或有明确系统限制和手工替代证据；
- API 解析测试覆盖正常、缺字段、破损 JSON；
- 缓存测试覆盖读写、损坏、网络失败不覆盖缓存；
- ViewModel 测试覆盖首页和详情核心状态；
- UI 冒烟测试覆盖阅读主链路；
- 测试代码不依赖真实外部接口，必须通过 mock fixture、`MockURLProtocol`、launch argument 或可控测试环境运行。

白盒测试最低覆盖：

| 模块 | XCTest 覆盖要求 |
| --- | --- |
| API | latest、before、detail 正常解析；非 2xx；破损 JSON；缺字段 |
| HomeViewModel | 加载成功、空态、刷新失败保留旧内容、历史加载、去重、缓存状态 |
| ArticleDetailViewModel | 详情成功、body 为空、分享链接缺失、失败读缓存、无缓存错误 |
| CacheStore | 写入、读取、损坏、30 天范围、失败不覆盖旧缓存 |
| 分享策略 | `share_url`、`detail.url`、`summary.url` 优先级 |

黑盒测试最低覆盖：

| 模块 | XCUITest 覆盖要求 |
| --- | --- |
| 首页主链路 | 启动、加载、顶部故事、普通列表 |
| 详情主链路 | 点击文章、展示详情、返回首页 |
| 分享入口 | 点击分享可出现系统分享面板或可验证入口状态 |
| 离线无缓存 | 首页错误态和重试入口 |
| 离线有缓存 | 首页/详情缓存提示 |
| 范围边界 | 不出现登录、评论、点赞、收藏、搜索等入口 |

### Step 4：主路径手工验收

手工执行：

```text
启动 App
→ 等待首页加载
→ 横向浏览顶部故事
→ 点击顶部故事进入详情
→ 返回首页
→ 点击普通列表文章进入详情
→ 点击分享
→ 取消分享
→ 返回首页
→ 向下加载历史日报
```

通过标准：

- 全链路无崩溃；
- 导航路径稳定；
- 文章正文可读；
- 分享面板可调起；
- 历史内容可追加；
- 返回首页后状态不丢失。

### Step 5：异常路径验收

必须覆盖：

| 场景 | 通过标准 |
| --- | --- |
| 无网络无缓存 | 首页显示错误态和重试入口 |
| 无网络有首页缓存 | 首页展示缓存内容和离线提示 |
| 无网络有详情缓存 | 详情展示缓存内容和缓存提示 |
| 无网络无详情缓存 | 详情展示错误态和重试入口 |
| 刷新失败 | 保留旧内容，提示“刷新失败，已保留上次内容” |
| 历史加载失败 | 保留已有列表，底部可重试 |
| 详情接口失败 | 有缓存读缓存，无缓存错误态 |
| JSON 解析失败 | 不崩溃，缓存兜底或错误态 |
| 缓存损坏 | 不崩溃，不白屏 |
| 正文为空 | 展示“文章内容暂不可用” |
| 分享链接缺失 | 不分享空内容，提示不可分享或禁用 |

### Step 6：视觉与体验验收

检查：

- 首页浅色模式可读；
- 首页深色模式可读；
- 详情外壳深色模式可读；
- 错误态、空态、离线提示中文清晰；
- 小屏 iPhone 不遮挡关键按钮；
- 长标题不撑破布局；
- 图片失败不影响文字阅读；
- WebView 不出现明显白屏。

### Step 7：范围边界验收

全局检查不得出现：

- 登录入口；
- 注册入口；
- 评论入口；
- 点赞入口；
- 收藏同步；
- 搜索；
- 主题日报；
- 推送通知；
- 小组件入口；
- 官方知乎 Logo；
- App Store 上架承诺。

出现任一项，应判定为范围失控，至少为 P1；若影响主流程或合规风险明显，可判为 P0。

## 8. 缺陷分级

| 等级 | 定义 | 示例 | 处理 |
| --- | --- | --- | --- |
| P0 | 阻塞发布或验收 | 崩溃、白屏、主链路断、无缓存错误态缺失、错误分享空链接 | 必须修复后重新验收 |
| P1 | 核心体验问题 | 深色模式局部不可读、历史失败无重试、长标题严重挤压 | 原则上修复；可有条件通过需说明 |
| P2 | 边界或体验优化 | 大屏布局不够优雅、图片占位不够美观、轻微性能问题 | 可进入后续版本 |

缺陷记录格式：

```text
缺陷编号：
发现版本 / commit：
模块：
优先级：P0 / P1 / P2
复现步骤：
实际结果：
预期结果：
截图/日志：
责任 agent：
修复 commit：
回归结果：
```

## 11. 准出标准

允许进入下一阶段的最低标准：

- 所有 P0 用例通过；
- P0 用例已有 XCTest / XCUITest 覆盖，或存在明确手工替代说明；
- 无崩溃、白屏、主链路中断；
- 首页、详情、分享、历史加载主流程可用；
- 无网络有缓存、无网络无缓存均有正确表现；
- 刷新失败不清空旧内容；
- 网络失败不覆盖可用缓存；
- 缺正文、缺图片、缺分享链接均可降级；
- 深色模式基本可读；
- 不做项没有进入产品；
- 验收报告完整。

若存在 P1 问题但判定“有条件通过”，必须同时满足：

- 不影响 P0 主链路；
- 不造成数据误导；
- 不造成合规或品牌风险；
- 已记录缺陷编号和后续修复计划。

## 12. 最终签收

正式签收需要三方确认：

| 角色 | 确认内容 | 结论 |
| --- | --- | --- |
| 产品经理 agent | 需求范围和体验符合 PRD | 通过/不通过 |
| 测试 agent | XCTest / XCUITest / 手工测试执行完成，P0 无阻塞 | 通过/不通过 |
| 监工验收 agent | 独立验收通过，可进入下一阶段 | 通过/不通过 |

最终签收语：

```text
《日报阅读器》v1.0 在 commit {commit} 上完成验收。
验收结论：通过 / 有条件通过 / 不通过。
遗留问题：无 / 见缺陷清单。
下一步：进入工程实施下一阶段 / 退回修复。
```
