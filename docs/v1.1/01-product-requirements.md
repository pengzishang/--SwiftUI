# 《日报阅读器》产品需求文档 v1.1

## 0. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 日报阅读器 |
| 产品版本 | v1.1 |
| 目标平台 | iOS 17+ |
| 技术方向 | SwiftUI + WebKit |
| 数据源 | 候选知乎日报接口 `https://news-at.zhihu.com/api/4` |
| 当前定位 | 学习作品集，不代表知乎官方产品 |

## 1. 版本目标

v1.1 在 v1.0 的“日报阅读闭环”基础上，补齐轻量个人阅读管理能力，并优化首次启动、首次打开文章、离线缓存和阅读排版体验。

用户应能完成：

1. 打开 App，看到品牌化启动屏；
2. 快速进入首页并优先看到本地缓存内容；
3. 浏览日报、收藏、已读记录；
4. 将文章移入冷宫、收藏或标为已读；
5. 在设置中调整文章正文字体和列表字体；
6. 进入详情页时隐藏 TabBar，阅读正文并点击正文图片全屏预览；
7. 在弱网或断网时尽量读取已缓存的首页与文章详情；
8. 首次打开文章详情时不出现明显 WebView 冷启动顿卡。

## 3. 功能范围

### 3.1 阅读管理

- 日报首页过滤已进入冷宫、收藏、已读的文章。
- 收藏页展示本地收藏文章。
- 已读页展示本地阅读历史，按最近阅读时间倒序排列。
- 冷宫页作为低频管理能力，放在设置页入口中。
- 一篇文章只出现在一个主要归属中，优先级为：冷宫 > 收藏 > 已读 > 日报。

### 3.2 详情页操作菜单

详情页右上角使用系统 `Menu`，根据来源展示不同操作：

- 分享；
- 收藏 / 取消收藏；
- 设为已读 / 设为未读；
- 不感兴趣；
- 从冷宫恢复到日报。

非已读入口默认只提供“设为已读”；已读入口提供“设为未读”。

### 3.3 设置

设置页提供：

- 文章字体大小：14 / 16 / 18 / 20 / 22；
- 列表字体大小：14 / 16 / 18 / 20 / 22；
- 冷宫入口；
- 当前版本号。

字体设置使用本地持久化，重启 App 后保持。

### 3.4 文章阅读增强

- 详情页进入时隐藏底部 TabBar。
- 导航栏标题显示文章标题。
- 正文 WebView 支持动态字体大小。
- “查看知乎讨论”按钮显示为满宽圆角按钮。
- `blockquote` 使用左侧灰色竖线和次级文字色。
- 正文图片点击后进入全屏预览，支持双指缩放和双击缩放。

### 3.5 缓存与弱网

- 首页完整 Feed（`sections` + `topStories`）优先读缓存展示，再后台刷新网络。
- 历史日报按业务日期缓存，网络失败时尝试读取对应日期缓存。
- 文章详情 cache-first：本地已有详情时直接展示，避免重复网络请求。
- 网络失败时保留已有内容，不用空页面覆盖可读内容。

### 3.6 启动与性能

- 使用 `UILaunchScreen` 替代旧 Launch Image。
- 启动页使用动态蓝色背景和居中品牌图。
- App 进入后延迟预热轻量 `WKWebView`，削减第一次打开文章的 WebKit 冷启动顿卡。
- `HTMLWebView.updateUIView` 避免重复拼接完整 HTML，并使用轻量 key 判断是否需要 reload。

## 4. 明确不做项

v1.1 仍不做：

- 登录、注册；
- 评论、点赞；
- 收藏云同步；
- 推送、小组件；
- 自定义分享渠道；
- App 内主题切换；
- 官方知乎日报视觉 1:1 复刻；
- 商业发布承诺。

## 5. 验收标准

- 首次启动展示非 deprecated 的 `UILaunchScreen` 启动页。
- 首页可从缓存快速恢复，并在网络成功后刷新。
- 日报、收藏、已读、设置四个 Tab 可正常切换。
- 冷宫可从设置页进入。
- 左滑操作和详情菜单可正确改变文章归属。
- 已读页支持搜索，并按最近阅读排序。
- 字体大小调整能影响文章正文和列表。
- 详情页图片点击可全屏预览和缩放。
- 首次打开文章不出现明显顿卡。
- `xcodebuild` 构建通过。
